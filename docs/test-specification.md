# テスト仕様書

## 1. テスト方針

MVPでは、業務ロジックを自動テストで確認し、主要画面の動作はブラウザ確認で補完する。GitHub Pages公開前に `test`、`typecheck`、`lint`、`build` を必ず実行する。

## 4. 手動確認観点

- 住所や緯度経度のような専門的な入力を営業担当者に強制していないか
- 訪問予定が180件規模になっても、地図上でまとめて選べる流れになっているか
- 管理者が朝一番に見るべき未割当、訪問NG、期限超過を把握できるか
- 管理者がKPI、未決定理由、システム管理へ迷わず移動できるか
- 未決定理由が単なるメモではなく、対策案、優先度、採用 / 未採用理由、周知まで扱えるか
- 紙地図の置き換えとして、徒歩負担や確認負担の削減につながるか
- 将来AWS、Cognito、DynamoDB、S3、地図APIへ差し替えやすい構成か

## 5. 合格条件

- 自動テストがすべて成功する
- TypeScriptチェックが成功する
- lintが成功する
- GitHub Pages向けビルドが成功する
- 管理者・営業担当者の主要ブラウザ確認で重大な表示崩れ、操作不能、コンソールエラーがない
